# Claude Configuration for spaiOS

Follow the coding standards and guidelines defined in [rules/CODING_STANDARDS.md](../rules/CODING_STANDARDS.md).

These guidelines apply to all work: features, bug fixes, refactoring, documentation, and tests.

## Key Principles

- **Small, meaningful commits** — one logical change per commit
- **Always test** — verify code before committing
- **High code quality** — readable, maintainable, follows conventions
- **Clear communication** — explain the "why" in code and commits

See [rules/CODING_STANDARDS.md](../rules/CODING_STANDARDS.md) for detailed standards.
