# WP2.4 — Named Specialised Subagent Profiles

**Date:** 2026-07-21  
**Status:** Implemented  
**Branch:** `wp2-4-named-subagents`

---

## Problem

The existing `delegate` tool spawns a single, generic depth-1 sub-agent. Every
delegation uses the same system prompt and the same full tool set as the parent,
giving the model no way to signal _what kind_ of work is being farmed out. A
reviewer sub-agent and a tester sub-agent are identical today; both can write
files and run destructive commands, which is more capability than most delegated
tasks need.

---

## Goals

1. Let the parent model name a **profile** when delegating (e.g. `reviewer`,
   `tester`).
2. Each profile carries its own **system prompt** and **tool allowlist** that
   narrow the sub-agent's surface.
3. Zero-config: ship sensible built-in profiles; user config extends or
   overrides.
4. **Never widen permissions**: a profile may only _restrict_ the sub-agent to a
   subset of the parent's tools, never grant tools the parent lacks.
5. Preserve the existing permission gate: the sub-agent re-gates each of its
   own actions through the parent's `confirmFn`, unchanged.

---

## Non-Goals

- Multiple delegation depths (still hard-limited to 1).
- Per-profile model selection (all profiles use the parent's provider).
- Dynamic profile creation at runtime.

---

## Design

### Prefer a single `delegate` tool with an added `profile` arg

The brief explicitly recommends this approach to avoid touching `classify()`. A
new tool name would require a new `classify()` case or fall through to the
`TierRead` default — a permission regression. Adding an optional `profile` field
to the existing `delegate` input keeps `classify()` unchanged (still
`TierWrite`) and is fully backwards-compatible (omitting `profile` gives the old
generic behaviour).

### Config schema (`[[subagent.profiles]]`)

```toml
[[subagent.profiles]]
name        = "reviewer"
description = "Read-only code reviewer."
system_prompt = "You are a precise code reviewer…"
tools       = ["read_file", "grep", "glob", "list_dir", "web_search", "web_fetch"]

[[subagent.profiles]]
name        = "tester"
description = "Test writer and runner."
system_prompt = "You are a software testing expert…"
tools       = ["bash", "read_file", "write_file", "edit_file", "glob", "grep", "list_dir"]
```

Parsed into `config.SubagentConfig.Profiles []SubagentProfile` under the
`[subagent]` TOML key.

### Built-in profiles (zero-config defaults)

| Name       | System prompt | Tool allowlist |
|------------|---------------|----------------|
| `reviewer` | Code-review focused read-only analyst | `read_file`, `grep`, `glob`, `list_dir`, `web_search`, `web_fetch` |
| `tester`   | Testing expert | `bash`, `read_file`, `write_file`, `edit_file`, `glob`, `grep`, `list_dir` |
| `general`  | Default system prompt | all parent tools (empty allowlist = no restriction) |

User config takes precedence over builtins when names collide.

### Data flow

```
parent Agent.NewWithRegistry()
  └─ registers delegate tool with closure:
       func(ctx, task, profile) → runDelegate(…, task, profile)

runDelegate(ctx, provider, confirmFn, childConfig, task, profile)
  1. resolve profile from child.SubagentProfiles ++ builtinProfiles
  2. filter registry: parentReg.Filter(profile.Tools)   ← never adds new tools
  3. if profile.SystemPrompt != "": child.SystemPromptOverride = profile.SystemPrompt
  4. build nested Agent with filtered registry + child config
  5. run nested loop, collect summary, return to parent
```

### Permission gate preservation

- `classify("delegate")` still returns `TierWrite` — no change to gating.
- The filtered registry physically cannot contain tools outside the parent set.
- The nested agent's `confirmFn` is the same unmodified parent function — every
  sub-action is gated identically to a top-level call.
- The delegate tool itself is absent from the nested registry — depth cannot
  exceed 1.

### Key new types / functions

| Location | Symbol | Purpose |
|----------|--------|---------|
| `config` | `SubagentProfile`, `SubagentConfig` | TOML-parsed profile definition |
| `agent`  | `SubagentProfile` | Runtime profile used by agent package |
| `agent`  | `Config.SubagentProfiles` | Resolved list passed to child config |
| `agent`  | `Config.SystemPromptOverride` | Profile system prompt for sub-agent loop |
| `agent`  | `resolveProfile()` | Lookup: user profiles first, then builtins |
| `tools`  | `DelegateRunner` signature | Extended to `func(ctx, task, profile string)` |
| `tools`  | `Delegate.Schema()` | Added optional `profile` property |
| `tools`  | `ProfileArg()` | Extract `profile` field from raw tool input |
| `tools`  | `Registry.Filter()` | Return subset registry for profile allowlist |
